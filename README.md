# Photo Audit Workspace (摄影平台供稿审核后台)

## 📖 项目简介
本项目是一个为现代摄影平台设计的**供稿审核后台 MVP (Minimum Viable Product)**。通过引入“AI 机审 + 人工复核”的工作流，彻底改变传统的人工审核模式。前端采用纯 HTML + Tailwind CSS + 原生 JavaScript 构建，提供极佳的暗黑模式沉浸式体验。

## 🎯 解决的业务问题
随着 UGC（用户生成内容）的爆发，摄影平台每天接收海量的图片供稿，传统纯人工审核面临三大痛点：
1. **效率低下**：海量图片的初筛耗费大量人力，出审时效长，影响创作者体验。
2. **风险漏判**：人工审核容易疲劳，导致版权争议、政治敏感或违规内容漏放。
3. **标准不一**：不同审核员对“画面模糊”、“过曝/欠曝”等技术瑕疵的评判主观性太强。

本系统旨在打造一个**高效、标准、安全**的审核工作台，让 AI 拦截掉 90% 的低质与违规稿件，让人工只专注于 10% 的边缘争议内容（Edge Cases）。

## 📊 核心指标设计
为了衡量 AI 赋能审核的业务价值，本项目设计了以下核心指标：
- **机审召回率 (Recall Rate)**：**> 99.2%**。针对“政治敏感”、“版权争议”等高风险内容，模型策略偏向“宁可错杀，不可放过”，确保风险内容全部进入人工复核队列，保证平台合规安全。
- **机审准确率 (Accuracy Rate)**：**> 85.0%**。针对“画面模糊”、“噪点过高”等低质量废片，要求模型有较高的准确率，直接执行系统自动打回，极大降低人工二次确认的成本。
- **平均出审时效 (Average Processing Time)**：从传统的分钟级缩短至 **1.2s** 以内。

## 🤖 AI 接入方案
本 Demo 已从静态假数据升级为**真实 AI 接入**，支持双模型自由切换：

| 模型 | 类型 | 接口端点 | 视觉能力 | 适用场景 |
|------|------|----------|----------|----------|
| **Gemini 2.0 Flash** | Vision | `generativelanguage.googleapis.com` | ✅ 原生支持图片输入 | 真正的图片内容审核（安全、质量、版权） |
| **DeepSeek Chat** | 纯文本 | `api.deepseek.com/chat/completions` | ❌ 基于元数据推理 | 文件格式/体积驱动的智能盲审（格式分析、版权推断） |

### 架构亮点
- **统一分发层**：`callAIForReview()` 根据当前 provider 自动路由到对应 API
- **额度耗尽检测**：自动识别 HTTP 402/429 及 API 返回的 quota/balance 错误关键词
- **一键切换弹窗**：额度耗尽时弹出琥珀色 UI 提示，引导用户无缝切换至另一模型
- **API Key 安全**：不写死在代码中，通过页面顶部横幅输入，全程浏览器内存中

### DeepSeek 盲审规则
由于 DeepSeek 不支持视觉输入，系统会将图片的格式/体积转化为元数据，结合以下规则交给 AI 判定：

| 格式 | 文件大小 | AI 判定 | 典型分数 |
|------|---------|---------|----------|
| PNG | < 50KB | `is_safe=false`，疑似截图/盗图 | 30-45 |
| PNG | 50-200KB | 边缘可疑，建议人工复核 | 50-65 |
| RAW/TIFF | > 5MB | 专业级原始文件，高质量 | 85-95 |
| JPEG | > 2MB | 优质投稿 | 75-90 |
| JPEG | < 200KB | 严重压缩，画质损失 | 45-65 |

## ✨ 核心功能
- **沉浸式暗黑工作台**：专为影像工作者设计的 Dark Mode UI
- **AI 智能辅助**：图片卡片自动展示 AI 机审建议（建议通过 / 需人工复核 / 建议打回），右上角显示评分徽章，hover 展示审核理由
- **双模型自由切换**：Gemini（视觉审核）/ DeepSeek（元数据盲审），一键切换，无需改代码
- **实时真实上传**：点击或拖拽真实图片文件，AI 自动完成审核并动态创建审核卡片
- **快捷打回**：Reject 时弹出细分原因菜单（版权争议、画面模糊、政治敏感）
- **摄影师申诉闭环**：打回后 Toast 附带 "Appeal" 按钮 → 弹出申诉浮层 → 提交后审核日志联动记录
- **业务实时看板**：侧边栏展示今日已审、平均出审时效、机审召回率
- **审核日志**：实时记录每一次 Approve/Reject/申诉操作，带时间戳，刷新不丢失
- **本地状态持久化**：localStorage 存储已审卡片、今日统计、审核日志

## 💻 本地运行
无需安装任何依赖，直接双击打开即可体验：
```bash
open index.html
```

### 接入真实 AI（可选）
1. 打开页面，顶部琥珀色横幅选择模型（Gemini / DeepSeek）
2. 前往对应平台获取免费 API Key：
   - Gemini：[Google AI Studio](https://aistudio.google.com/apikey)
   - DeepSeek：[DeepSeek Platform](https://platform.deepseek.com/api_keys)
3. 粘贴 Key 并点击 "启用 AI"
4. 点击或拖拽任意图片到上传区，等待 AI 返回审核结果

> 未配置 API Key 时，上传区默认使用假数据模拟，所有静态卡片仍可正常审核。
