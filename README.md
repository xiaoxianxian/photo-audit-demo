# Photo Audit Workspace (摄影平台供稿审核后台)

## 📖 项目简介
本项目是一个为现代摄影平台设计的**供稿审核后台 MVP (Minimum Viable Product)**。通过引入“AI 预审 + 人工复核”的工作流，彻底改变传统的人工审核模式。前端采用纯 HTML + Tailwind CSS + 原生 JavaScript 构建，提供极佳的暗黑模式沉浸式体验。

## 🎯 解决的业务问题
随着 UGC（用户生成内容）的爆发，摄影平台每天接收海量的图片供稿，传统纯人工审核面临三大痛点：
1. **效率低下**：海量图片的初筛耗费大量人力，出审时效长，影响创作者体验。
2. **风险漏判**：人工审核容易疲劳，导致版权争议、政治敏感或违规内容漏放。
3. **标准不一**：不同审核员对“画面模糊”、“过曝/欠曝”等技术瑕疵的评判主观性太强。

本系统旨在打造一个**高效、标准、安全**的审核工作台，让 AI 拦截掉 90% 的低质与违规稿件，让人工只专注于 10% 的边缘争议内容（Edge Cases）。

## 📊 核心指标设计
为了衡量 AI 赋能审核的业务价值，本项目设计了以下核心指标：
- **机审召回率 (Recall Rate)**：**> 99.2%**。针对“政治敏感”、“版权争议”等高风险内容，模型策略偏向“宁可错杀，不可放过”，确保风险内容全部进入人工复核队列，保证平台合规安全。
- **机审准确率 (Accuracy Rate)**：**> 85.0%**。针对“画面模糊”、“噪点过高”等低质量废片，要求模型有较高的准确率，直接执行系统自动打回，极大降低人工二次确认的成本。
- **平均出审时效 (Average Processing Time)**：从传统的分钟级缩短至 **1.2s** 以内。

## 🤖 规划的 AI 技术栈
虽然本 Demo 是前端交互原型，但在真实的生产环境中，背后的 AI 引擎规划如下：
1. **图像质量评估 (IQA)**：基于 **ViT (Vision Transformer)** 微调的图像质量评分模型，识别失焦、噪点、构图问题。
2. **版权溯源与查重**：基于 **CLIP** 的多模态特征提取，结合 Milvus 向量数据库，实现海量图库的“以图搜图”和侵权比对。
3. **敏感内容拦截 (NSFW & Risk Detection)**：基于 **ResNet** 及目标检测算法，过滤涉黄、涉暴、涉政内容。
4. **OCR 文字识别**：提取图片中可能包含的违规水印、广告或敏感文字。

## ✨ 核心功能
- **沉浸式暗黑工作台**：专为影像工作者设计的 Dark Mode UI。
- **AI 智能辅助**：图片卡片自动展示 AI 机审建议（如：建议通过、疑似版权风险、画面模糊）。
- **快捷操作与快捷打回**：支持快速 Approve，并在 Reject 时提供细分的原因（版权争议、画面模糊、政治敏感）。
- **摄影师申诉闭环**：打回操作后支持一键唤起申诉窗口，完善创作者体验。
- **本地状态持久化**：使用 `localStorage` 模拟后端数据库，实现刷新不丢失的业务数据（今日已审统计、审核日志）。
